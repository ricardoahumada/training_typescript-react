1. El Compilador TypeScript

¿Qué es tsc?

¿Qué es tsc?

tsc = TypeScript Compiler

  • Transforma código TypeScript a JavaScript estándar
  • Elimina los tipos en tiempo de compilación
  • Genera archivos .js y opcionalmente .js.map

Proceso de Compilación

archivo.ts ──► tsc ──► archivo.js ──► Node.js / Navegador

Flujo: TypeScript → Compilador → JavaScript → Ejecución

¿Por Qué Compilar?

Razón Explicación
Compatibilidad Los navegadores no entienden TypeScript, solo JavaScript
Tamaño El código compilado es más pequeño (sin tipos)
Portabilidad Compatible hacia atrás con cualquier entorno JS

2. tsconfig.json

Guía Completa de Configuración

¿Qué es tsconfig.json?

  • Archivo de configuración del compilador TypeScript
  • Define cómo se compila el código
  • Personaliza el comportamiento de tsc

Ubicación: Raíz del proyecto TypeScript

Estructura Básica

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

Opción target: Versión de JavaScript

Valor Descripción Compatibilidad
ES3 JavaScript más antiguo IE6+
ES5 JavaScript moderno Todos los navegadores
ES2015 / ES6 ES6 Navegadores modernos
ES2020 ES2020 Navegadores recientes
ESNext Última versión Entornos modernos

Opciones Principales

Opción Descripción
target Versión de JavaScript destino
module Sistema de módulos (commonjs, esnext)
outDir Carpeta de salida
rootDir Carpeta raíz del código fuente
strict Habilitar modo estricto
lib Bibliotecas incluidas
esModuleInterop Interoperabilidad entre módulos
skipLibCheck Omitir verificación de tipos en librerías
declaration Generar archivos .d.ts
sourceMap Generar source maps

Archivos a Incluir/Excluir

{
  "compilerOptions": { ... },
  
  // Archivos a incluir en la compilación
  "include": [
    "src/**/*",      // Todo en src/
    "*.ts"           // Archivos .ts en raíz
  ],
  
  // Archivos a excluir de la compilación
  "exclude": [
    "node_modules",  // Dependencias
    "dist",         // Salida anterior
    "**/*.test.ts", // Tests
    "*.spec.ts"
  ]
}

Conclusión: tsconfig.json

  • tsconfig.json es el corazón de la compilación
  • Configura target según tu entorno destino
  • Usa strict: true para mejor calidad de código
  • include y exclude controlan qué archivos se compilan

Lab

Configuración y Compilación

Ejercicio Práctico

Objetivo: Configurar un proyecto TypeScript y compilarlo a JavaScript

Continuaremos con el Caso 1: Sistema de Gestión de Tareas

Pasos para el Lab

  1. Crear archivo tsconfig.json con configuración básica
  2. Configurar target: ES2020 y module: commonjs
  3. Establecer outDir: "./dist" y rootDir: "./src"
  4. Habilitar strict: true y sourceMap: true
  5. Ejecutar npx tsc para compilar
  6. Verificar los archivos generados en dist/

Configuración Esperada

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "declaration": true,
    "sourceMap": true,
    "lib": ["ES2020"]
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

Verificación del Lab

  • Ejecutar npx tsc sin errores
  • Archivos .js generados en dist/
  • Archivos .js.map generados
  • Archivos .d.ts generados (si aplica)

¡Excelente! Tu proyecto ahora compila correctamente

4. Scripts de Compilación

Automatiza tu flujo de trabajo

Scripts en package.json

{
  "scripts": {
    "build": "tsc",
    "dev": "tsc --watch",
    "start": "node dist/index.js",
    "clean": "rm -rf dist"
  }
}

Ejecutar Compilación

  • Compilar una vez: npm run build
  • Modo vigilancia: npm run dev
  • Ejecutar: npm start

Tips y Gotchas

Errores comunes y mejores prácticas

✅ Tips

  • Usa strict: true desde el inicio
  • target debe coincidir con tus requisitos
  • outDir diferente de rootDir
  • declaration genera .d.ts para paquetes

⚠️ Gotchas

  • Cambios en tsconfig no se recargan - reinicia VS Code
  • exclude no garantiza exclusión total
  • moduleResolution: "node" vs "bundler"
  • tsc -w no detecta nuevos archivos
