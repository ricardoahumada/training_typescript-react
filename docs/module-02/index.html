1. VS Code

El editor profesional

Por qué VS Code?

Ventajas

  • Gratis y open source
  • Multiplataforma: Windows, macOS, Linux
  • IntelliSense: autocompletado inteligente
  • Extensiones: ecosistema huge

Características

  • Terminal integrada
  • Integración con Git
  • Depuración integrada
  • Live Server

Configuración recomendada

{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "typescript.preferences.includePackageJsonAutoImports": "on",
  "files.autoSave": "afterDelay",
  "eslint.validate": ["typescript", "typescriptreact"]
}

Abre settings.json con Ctrl+Shift+P > "Open Settings (JSON)"

2. Instalación en Linux

Configuración completa

Instalar VS Code

Debian/Ubuntu

sudo apt update
sudo apt install wget gpg
wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -D -o root -g root -m 644 packages.microsoft.gpg /etc/apt/keyrings/packages.microsoft.gpg
sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
sudo apt update
sudo apt install code

Fedora/RHEL

sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'
sudo dnf check-update
sudo dnf install code

3. Git para el Proyecto

Control de versiones

Comandos básicos

# Inicializar repositorio
git init

# Ver estado
git status

# Stage todos los archivos
git add .

# Commit con mensaje
git commit -m "feat: primer commit"

# Ver historial
git log

Flujo de trabajo con ramas

  • main: Código estable
  • develop: Desarrollo
  • feature/nombre: Nueva funcionalidad
  • git checkout -b feature/nueva-rama

.gitignore esencial

# Node.js
node_modules/
dist/
npm-debug.log*

# TypeScript
*.tsbuildinfo

# Entorno
.env
.env.local

# IDE
.vscode/
.idea/
*.swp
*.swo

# OS
.DS_Store
Thumbs.db

# Logs
*.log
logs/

Crea .gitignore desde el inicio del proyecto

4. npm y Dependencias

Gestión de paquetes

package.json

{
  "name": "mi-proyecto-typescript",
  "version": "1.0.0",
  "description": "Mi primer proyecto TypeScript",
  "scripts": {
    "build": "tsc",
    "dev": "tsc --watch"
  },
  "devDependencies": {
    "typescript": "^5.3.0"
  }
}

Comandos npm esenciales

  • npm init -y - Inicializar proyecto
  • npm install paquete - Dependencia de producción
  • npm install -D paquete - Dependencia de desarrollo
  • npm run build - Ejecutar script de build

5. Primera Prueba de Funcionamiento

Verificar el entorno

# 1. Verificar Node.js
node --version

# 2. Verificar npm
npm --version

# 3. Verificar Git
git --version

# 4. Verificar TypeScript
tsc --version

# 5. Verificar VS Code
code --version

Crear primer proyecto TypeScript (Caso 1)

# 1. Crear carpeta del proyecto
mkdir task-manager
cd task-manager

# 2. Inicializar npm
npm init -y

# 3. Instalar TypeScript
npm install -D typescript

# 4. Inicializar tsconfig
npx tsc --init

# 5. Crear archivo TypeScript
echo 'const greeting: string = "Hola TypeScript";' > index.ts

# 6. Compilar
npx tsc

# 7. Ejecutar
node index.js

¡Felicidades! Has creado tu primer proyecto TypeScript.
