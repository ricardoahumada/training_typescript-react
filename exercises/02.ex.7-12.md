# Ejercicio Integrador 2: Sistema de Biblioteca - Repaso Módulos 7-12

## Caso 2: Sistema de Gestión de Biblioteca

## Objetivo
Extender el sistema de biblioteca con funcionalidades avanzadas utilizando TypeScript,
aplicando conceptos de los módulos 7-12: funciones tipadas, clases, genéricos, módulos y testing.

## Duración
60-75 minutos (trabajo en grupo de 2-4 personas)

## Requisitos Previos
- Módulos 7-12 completados
- Ejercicio integrador 1 (Módulos 1-6) completado
- Conceptos: funciones tipadas, clases, herencias, genéricos, módulos, testing

---

## Enunciado

### Contexto
El sistema de biblioteca está funcionando bien, pero ahora necesitas añadir
validación avanzada, operaciones masivas, y coverage de tests para asegurar calidad.

### Tu Misión
Implementar servicios avanzados y tests para el sistema de biblioteca.

---

## Parte 1: Servicio de Préstamos con Overloads (Módulo 7)

El servicio de préstamos necesita manejar diferentes escenarios:

**Situación 1**: El sistema debe procesar préstamos con distinta duración según el tipo de socio.
- Préstamo estándar: aplica días por defecto según membresía
- Préstamo extendido: permite especificar días custom
- Préstamo prioritario: marca el préstamo con prioridad y días custom

**Situación 2**: El sistema necesita notificar cuando un préstamo se completa.
- Definir un tipo callback que reciba: el préstamo creado, si tuvo éxito, y un mensaje
- Implementar una función de renovación que sea asíncrona y lance errores tipados

**Preguntas a resolver**:
- ¿Cómo defines múltiples firmas para una misma función?
- ¿Cómo tipas un callback como parámetro?
- ¿Cómo tipas una función asíncrona con errores específicos?

### Deliverable 1
Archivo src/services/loanService.ts con overloads de función, callback tipado y función async.

---

## Parte 2: Jerarquía de Clases (Módulo 8)

El catálogo de la biblioteca incluye tanto libros físicos como digitales.

**Situación 1**: Necesitas una jerarquía de clases para representar los items del catálogo.
- Clase base: debe tener id, timestamps de creación y actualización, métodos para actualizar y serializar
- Libro físico: hereda de la base y añade isbn, título, autores, género
- Libro digital: hereda de libro físico y añade url y formato. El nombre a mostrar debe indicar que es digital

**Situación 2**: Los items deben ser buscables.
- Definir una interfaz Searchable con un método search que recibe query string
- Implementar la interfaz en libro físico y digital

**Preguntas a resolver**:
- ¿Cómo implementas herencia de clases en TypeScript?
- ¿Cómo usas modificadores de acceso (protected, private)?
- ¿Cómo haces override de un método?
- ¿Cómo implementas una interfaz en una clase?

### Deliverable 2
Archivo src/models/LibraryItem.ts con la jerarquía completa.

---

## Parte 3: Utilidades Genéricas (Módulo 9)

Necesitas funciones reutilizables para operar sobre colecciones del sistema.

**Situación 1**: Filtrar colecciones por predicado.
- Crear una función genérica que filtre un array usando una función predicado

**Situación 2**: Ordenar colecciones por clave.
- Crear función genérica que ordene por una clave específica, con orden ascendente o descendente

**Situación 3**: Persistencia básica de entidades.
- Crear una clase Repository genérica que gestione un CRUD básico de cualquier entidad con id
- Métodos: add, getById, getAll, remove, update

**Preguntas a resolver**:
- ¿Cómo defines un tipo genérico con constraints?
- ¿Cómo usas keyof para restringir claves?
- ¿Cómo tipas funciones de orden superior?

### Deliverable 3
Archivo src/utils/generics.ts con filterBy, sortBy y Repository<T>.

---

## Parte 4: Organización en Módulos (Módulo 10)

El código necesita restructurarse para seguir buenas prácticas de organización.

**Situación**: Reorganizar la estructura del proyecto usando módulos limpios.
- Crear la estructura de carpetas: types, models, services, repositories, utils
- Configurar paths en tsconfig.json para poder importar usando alias
- Crear un barrel exports (index.ts) que re-exporte todo

**Preguntas a resolver**:
- ¿Cómo configuras baseUrl y paths en tsconfig.json?
- ¿Cómo resuelves imports circulares?
- ¿Cómo implementas un barrel exports efectivo?

### Deliverable 4
Proyecto reorganizado con imports limpios usando paths configurados.

---

## Parte 5: Exports Modularizados (Módulo 11)

El sistema de imports/exports necesita ser consistente.

**Situación**: Asegurar que cada módulo exporta correctamente sus símbolos.
- Exports nombrados (named exports) para tipos y constantes
- Re-exports desde archivos index
- Default export para al menos una clase principal

**Preguntas a resolver**:
- ¿Cuándo usar export type vs export?
- ¿Cómo hacer re-exports limpios?
- ¿Cuándo es apropiado un default export?

### Deliverable 5
Verificar que todos los imports funcionan con la nueva estructura modular.

---

## Parte 6: Testing (Módulo 12)

El código necesita tests para garantizar calidad.

**Situación**: Implementar tests unitarios para las utilidades genéricas.
- Configurar Jest con ts-jest
- Escribir tests para filterBy que cubran: array vacío, un match, ningún match
- Escribir tests para sortBy que cubran: orden ascendente, orden descendente
- Escribir tests para Repository que cubran: add, getById, getAll, remove, update

**Preguntas a resolver**:
- ¿Cómo configuras Jest para TypeScript?
- ¿Cómo escribes tests que no tengan falsos positivos?
- ¿Qué hace un buen test unitario?

### Deliverable 6
Carpeta tests/ con tests unitarios ejecutables.

## Errores Comunes a Evitar

1. No usar any en ningún lugar
2. No olvidar el tipo de retorno en funciones
3. Evitar imports circulares entre módulos
4. Tests deben tener assertions significativas
5. Mantener strict mode activado en tsconfig.json
