# Ejercicio Integrador 1: Sistema de Biblioteca - Repaso Módulos 1-6

## Caso 2: Sistema de Gestión de Biblioteca

## Objetivo
Implementar las entidades base y lógica de negocio del Sistema de Biblioteca
utilizando TypeScript puro, aplicando conceptos de los módulos 1-6.

## Duración
45-60 minutos (trabajo en grupo de 2-4 personas)

## Requisitos Previos
- Módulos 1-6 completados
- Conceptos: tipos básicos, interfaces, variables, compilación, type guards

---

## Enunciado

### Contexto
Eres parte del equipo de desarrollo de una biblioteca pública que necesita
modernizar su sistema de gestión de préstamos. El sistema actual es legacy
(y te show code más adelante) y han decidido reescribirlo en TypeScript.

### Tu Misión
Implementar la capa de tipos y validación del nuevo sistema de biblioteca.

---

## Parte 1: Definición de Entidades (Módulo 1 - Intro a TS)

Definir las interfaces TypeScript para las entidades del dominio:

1. **Book** - Libro en el catálogo
   - ISBN (string único)
   - Título (string)
   - Autor(es) - array de strings
   - Género (enum: FICTION, NON_FICTION, SCIENCE, HISTORY, TECHNOLOGY, BIOGRAPHY, CHILDREN)
   - Año de publicación (number)
   - Editorial (string)
   - Ejemplares disponibles (number)
   - Total de ejemplares (number)
   - Formato digital (boolean)

2. **Member** - Socio de la biblioteca
   - ID (string)
   - Nombre (string)
   - Email (string)
   - Teléfono (string)
   - Tipo de membresía (enum: STANDARD, PREMIUM, STUDENT)
   - Fecha de registro (Date)
   - Préstamos activos (number)
   - Máximo de préstamos permitidos (number)

3. **Loan** - Préstamo de libro
   - ID (string)
   - ISBN del libro (string)
   - ID del socio (string)
   - Fecha de préstamo (Date)
   - Fecha de devolución esperada (Date)
   - Fecha de devolución real (Date | undefined)
   - Estado del préstamo (enum: ACTIVE, RETURNED, OVERDUE, LOST)

### Deliverable 1
Archivo `src/types/index.ts` con todas las interfaces y enums definidos.

## Parte 3: Lógica de Negocio - Tipado de Funciones (Módulo 5 - Variables y Tipado)

Implementar las siguientes funciones con tipado completo:

1. **createBook(data)**: Crea un libro validado
   - Parámetro: objeto parcial de Book
   - Retorna: Book completo
   - Validaciones:
     - ISBN debe tener formato válido (13 caracteres numéricos)
     - Título no puede estar vacío
     - Ejemplares disponibles no puede superar total
     - Año de publicación no puede ser futuro

2. **registerMember(data)**: Registra un nuevo socio
   - Parámetro: objeto parcial de Member
   - Retorna: Member completo
   - Validaciones:
     - Email debe tener formato válido
     - Teléfono debe tener al menos 9 dígitos
     - Límite de préstamos según tipo de membresía:
       - STANDARD: 3 libros
       - PREMIUM: 10 libros
       - STUDENT: 5 libros

3. **calculateDueDate(loanDate, membershipType)**: Calcula fecha de devolución
   - STANDARD: 14 días
   - PREMIUM: 30 días
   - STUDENT: 21 días

### Deliverable 3
Archivo `src/services/library.ts` con las tres funciones implementadas.

## Parte 5: Type Guards y Validación (Módulo 6 - Estructuras de Control)

Implementar type guards para validación robusta:

1. **isValidISBN(isbn: unknown)**: boolean
   - Verifica que es string
   - Verifica que tiene exactamente 13 caracteres
   - Verifica que todos son dígitos

2. **isValidEmail(email: unknown)**: boolean
   - Verifica formato básico de email

3. **isBook(obj: unknown)**: obj is Book
   - Usa type guard para validar estructura de Book

4. **isMember(obj: unknown)**: obj is Member
   - Usa type guard para validar estructura de Member

### Deliverable 5
Archivo `src/utils/validators.ts` con todos los type guards.

## Verificación Global

Para verificar que todo funciona correctamente, el siguiente código debe compilar y ejecutarse sin errores:

```typescript
import { Book, Member, Loan, Genre, MembershipType, LoanStatus } from './types';
import { createBook, registerMember, calculateDueDate } from './services/library';
import { isValidISBN, isBook, isMember } from './utils/validators';

// Test 1: Crear libro válido
const book = createBook({
  isbn: '9780123456789',
  title: 'El Quijote',
  authors: ['Miguel de Cervantes'],
  genre: Genre.FICTION,
  publicationYear: 1605,
  publisher: 'Juan de la Cuesta',
  totalCopies: 5,
  availableCopies: 3,
  isDigital: false
});
console.log('Libro creado:', book);

// Test 2: Registrar socio
const member = registerMember({
  name: 'María García',
  email: 'maria@email.com',
  phone: '612345678',
  membershipType: MembershipType.STANDARD
});
console.log('Socio registrado:', member);

// Test 3: Calcular fecha de devolución
const dueDate = calculateDueDate(new Date(), member.membershipType);
console.log('Fecha de devolución:', dueDate);

// Test 4: Type guards
console.log('ISBN válido:', isValidISBN('9780123456789'));
console.log('Es un libro válido:', isBook(book));
console.log('Es un socio válido:', isMember(member));

// Test 5: Préstamo
const loan: Loan = {
  id: 'LOAN-001',
  bookIsbn: book.isbn,
  memberId: member.id,
  loanDate: new Date(),
  dueDate: dueDate,
  returnDate: undefined,
  status: LoanStatus.ACTIVE,
  renewalCount: 0
};
console.log('Préstamo creado:', loan);
```

## Errores Comunes a Evitar

1. **No usar `any`** - Usar `unknown` para parámetros genéricos
2. **Olvidar el tipo de retorno** - Siempre especificar el return type
3. **No validar inputs** - Usar type guards antes de operar
4. **Enums mal definidos** - Usar strings, no números
5. **tsconfig muy laxo** - Mantener strict mode activado
