2. Utility Types

25 minutos

Partial, Required, Omit, Pick

UtilityUso
Partial<T>Todos los campos opcionales
Required<T>Todos los campos obligatorios
Omit<T, K>Excluir campos
Pick<T, K>Seleccionar campos
Record<K,V>Índice/Map

Por qué usar Utility Types - DRY

Problema: Definir tipos similares manualmente causa duplicación

// SIN utility types - duplicación!
interface TaskEdit { title?: string; ... }
interface TaskCreate { title: string; ... }

// CON utility types - DRY!
type TaskEdit = Partial<Task>;
type TaskCreate = Omit<Task, 'id'>;
BeneficioDescripción
Una fuente de verdadCambias Task, todo se actualiza
Menos bugsNo redefines tipos incorrectamente
MantenimientoUn cambio actualiza 5 tipos derivados

3. Generic Constraints

20 minutos

extends para limitar genéricos

function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const task = { title: 'Learn TS', priority: 'high' };
getProp(task, 'title');    // ✅ string
getProp(task, 'priority'); // ✅ string
getProp(task, 'status');   // ❌ Error: no existe

extends vs any - Compile Time Errors

any permite cualquier cosa - sin seguridad

// SIN constraint - any
function getValue(obj: any, key: any) {
  return obj[key]; // ✅ compila, ❌ puede fallar en runtime!
}

// CON extends - type safety
function getValue<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]; // ✅ compila SI y solo SI key existe
}
SituaciónSolución
Debe tener ciertas propsextends { prop: type }
Debe ser objetoextends object
Solo keys válidasextends keyof T

Lab: Generic Constraints

  1. Crear componente genérico con constraint

Código del Lab

interface WithId {
  id: string;
}

interface ListProps<T extends WithId> {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
  keyExtractor: (item: T) => string;

}

4. Template Literal Types

Crear tipos de strings específicas

La Magia: Errores en Compile Time

Template literals detectan errores antes de runtime

// CON template literals - error en compile time
type TaskEvent = 'onClick' | 'onSubmit';
type Handler = { [K in TaskEvent]?: () => void };
const h: Handler = { onClik: () => {} }; // ❌ Error!
CasoBeneficio
Rutas APIDetecta rutas inválidas

5. Mapped Types

15 minutos

Transformar tipos automáticamente

type Readonly<T> = { readonly [K in keyof T]: T[K] };
type Optional<T> = { [K in keyof T]?: T[K] };
type Nullable<T> = { [K in keyof T]: T[K] | null };

DRY: Transformaciones Automáticas

Mapped Types permiten transformar tipos sin duplicar código

// SIN Mapped Types - manual y propenso a errores
interface User { name: string; email: string; }
type ReadonlyUser = { readonly name: string; readonly email: string; }; // duplicado!

// CON Mapped Types - una fuente de verdad
type Readonly<T> = { readonly [K in keyof T]: T[K] };
type ReadonlyUser = Readonly<User>; // se actualiza si User cambia
TransformaciónUso
Readonly<T>Props que no deben mutar
Optional<T>Formularios de edición
Nullable<T>API responses

Lab: Mapped Types

  1. Hacer todos los campos opcionales
  2. Agregar prefijos

Código del Lab

// Mapped type para hacer todos los campos opcionales
type Partial = { [K in keyof T]?: T[K] };

// Mapped type para hacer todos los campos readonly
type Readonly = { readonly [K in keyof T]: T[K] };

// Mapped type para hacer todos los campos nullable
type Nullable = { [K in keyof T]: T[K] | null };

// Mapped type con prefijo
type WithPrefix = {
  [K in keyof T as `${P}${K & string}`]: T[K]
};

type TaskPrefix = WithPrefix;
// { task_id: string; task_title: string; ... }

6. Type Inference

10 minutos

Inferencia en funciones

// TS infiere el tipo de retorno
function getTask() {
  return { id: '1', title: 'Learn TS' };
}
// Infiere: { id: string; title: string }

// as const - preservar literales
const colors = ['red', 'blue'] as const;
// readonly ['red', 'blue']

Inferencia: Cuándo Confiar y Cuándo Ser Explícito

SituaciónInferenciaPor qué
Variables locales✅ SíTS infiere bien
Return de funciones✅ SíTS infiere el tipo exacto
Arrays de objetos✅ SíInfiere cada propiedad
Constantes literales❌ NoQuiere string, usa as const
// EXPLÍCITO cuando necesitas control
function createTask(title: string): Task { ... }

// as const - preserva literales
const STATUSES = ['pending', 'done'] as const;
type Status = typeof STATUSES[number]; // 'pending' | 'done'

7. Satisfies Operator

10 minutos

satisfies vs as

// as - pierde literal
const c1 = 'debug' as string; // c1: string

// satisfies - mantiene literal + valida
const c2 = 'debug' satisfies 'debug' | 'info' | 'error';
// c2: 'debug' (literal)

Decision Matrix: satisfies vs as

Necesitas...Solución
Validar tipo PERO mantener literalsatisfies
Forzar tipo sin importar el valoras
Autocomplete de literalessatisfies
Casting a tipo incompatibleas
// Caso práctico: validación con inferencia
type Color = 'primary' | 'secondary' | 'success';
const colores = {
  activo: 'primary',
  hover: 'secondary'
} satisfies Record<string, Color>;

// colores.activo es "primary" (literal), no string
usarColor(colores.activo); // ✅ Funciona

Lab: Operador Satisfies para el Caso 1r

// Caso: Configuración de tareas tipada
type Priority = 'low' | 'medium' | 'high';

const taskConfig = {
  maxTitleLength: 100,
  defaultPriority: 'medium' as const,
  allowedStatuses: ['pending', 'inProgress', 'completed'] as const
} satisfies {
  maxTitleLength: number;
  defaultPriority: Priority;
  readonly allowedStatuses: readonly (keyof Task)[];
};

// ✅ type narrowing: 'medium' literal, no string
type Priority2 = 'low' | 'medium' | 'high';
function setPriority(p: Priority2) {}
setPriority(taskConfig.defaultPriority); // ✅

8. Migración JSX → TSX

15 minutos

De JSX a TSX

// TSX - con tipos explícitos
interface BookCardProps {
  titulo: string;
  autor: string;
  año?: number;
}

function BookCard({ titulo, autor, año }: BookCardProps) {
  return (
    <div>
      <h3>{titulo}</h3>
      <p>{autor} {año && `- ${año}`}</p>
    </div>
  );
}

ROI: Por qué Migrar a TSX

AspectoJSXTSX
Errores de propsRuntimeCompile time ✅
AutocompleteNingunoTotal ✅
RefactoringPeligrosoSeguro ✅
DocumentaciónComentariosTipos ✅
// JSX - error en runtime!
<BookCard titulo="Libro" /> // ✅ Compila, ❌ falla (titulo vs title)

// TSX - error en compile time!
<BookCard titulo="Libro" /> // ❌ Error: 'titulo' does not exist

11. Tipado de Eventos

15 minutos

Eventos de formulario

// Input change
onChange={(e) => setValue(e.target.value)}
// e: ChangeEvent<HTMLInputElement>

// Form submit
onSubmit={(e) => e.preventDefault()}
// e: FormEvent<HTMLFormElement>

Eventos: Errores en Compile Time

// ❌ ERROR: type 'string' is not assignable
<input onChange={(e) => setValue(123)} />

// ✅ CORRECTO: type-safe
<input value={value} onChange={(e) => setValue(e.target.value)} />

// ❌ ERROR: Property 'target' does not exist
<button onClick={() => console.log(e.target)}>

// ✅ CORRECTO: tienes el tipo completo del evento
<button onClick={(e) => console.log(e.currentTarget)}>

function TaskForm({ onSubmit }: { onSubmit: (data: TaskFormData) => void }) {

Lab: Optimizar Componente con Hooks

function TaskList({ tasks, filter }: { tasks: Task[]; filter: string }) {
  const inputRef = useRef<HTMLInputElement>(null);

  const filteredTasks = useMemo(() => {
    if (!filter) return tasks;
    return tasks.filter(task => 
      task.title.toLowerCase().includes(filter.toLowerCase())
    );
  }, [tasks, filter]);

  const focusInput = useCallback(() => {

  );
}

Lab: Componente List Genérico

interface Entity { id: string; }
interface Task extends Entity { title: string; description: string; status: string; }

interface DataListProps<T extends Entity> {
  data: T[];
  renderItem: (item: T) => React.ReactNode;
  keyExtractor?: (item: T) => string;
}

function DataList<T extends Entity>({ data, renderItem, keyExtractor = (item) => item.id }: DataListProps<T>) {
  if (data.length === 0) return <div>No hay datos</div>;

  return (
    <ul>
      {data.map(item => (
        <li key={keyExtractor(item)}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}

  <>
    <label htmlFor="input">{item.name}</label>
    <input id="input" />
  </>
))}

// ✅ CORRECTO: useId genera IDs únicos por render
{items.map(item => {
  const id = useId(); // ⚠️ No dentro de map (hook rule)
  return (
    <>
      <label htmlFor={id}>{item.name}</label>
      <input id={id} />
    </>
  );
})}

Lab: IDs Únicos en Componentes

import { useId } from 'react';

interface InputProps {
  label: string;
  type?: 'text' | 'email' | 'password';
  value: string;
  onChange: (value: string) => void;
}

function Input({ label, type = 'text', value, onChange }: InputProps) {
  const id = useId();

  return (
    <div>
      <label htmlFor={id}>{label}</label>
      <input id={id} type={type} value={value} onChange={(e) => onChange(e.target.value)} />
    </div>
  );
}
