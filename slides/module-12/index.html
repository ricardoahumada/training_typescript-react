1. Contexto

Testing en TypeScript

Por qué Testing con Tipos

TypeScript mejora el testing:

// Jest con tipos
test("should create task", () => {
  const task = createTask("Test task", "high");
  
  // TypeScript sabe el tipo de task
  expect(task.title).toBe("Test task");
  expect(task.priority).toBe("high");
});

El autocompletado y la verificación de tipos hacen que los tests sean más confiables.

3. Tests Unitarios

Escribiendo Tests

Los tests unitarios prueban funciones de forma aislada.

// src/utils/calculator.ts
export function add(a: number, b: number): number {
  return a + b;
}

// tests/calculator.test.ts
import { add } from "../src/utils/calculator";

describe("Calculator", () => {
  test("adds two numbers", () => {
    expect(add(2, 3)).toBe(5);
  });
  
  test("handles negative numbers", () => {
    expect(add(-1, -2)).toBe(-3);
  });
});

Código del Lab

// tests/taskService.test.ts
import { TaskService } from "../src/services/taskService";

describe("TaskService", () => {
  let service: TaskService;
  
  beforeEach(() => {
    service = new TaskService();
  });
  
  describe("createTask", () => {
    it("should create a task with given title", () => {
      const task = service.createTask("New task", "high");
      
      expect(task.title).toBe("New task");
      expect(task.priority).toBe("high");
      expect(task.status).toBe("pending");
    });
    
    it("should generate unique id", () => {
      const task1 = service.createTask("Task 1", "low");
      const task2 = service.createTask("Task 2", "low");
      
      expect(task1.id).not.toBe(task2.id);
    });
  });
});

4. Mocks Tipados

Mocks con Tipo

Los mocks tipados facilitan testing de servicios dependientes.

// Mock de repositorio
const mockTaskRepository = {
  findAll: jest.fn(),
  findById: jest.fn(),
  save: jest.fn(),
  delete: jest.fn()
} as jest.Mocked<ITaskRepository>;

// Mock con implementación
const mockDate = new Date("2024-01-01");
jest.spyOn(global, "Date").mockImplementation(() => mockDate);

Lab: Mocks para el Sistema de Tareas

Objetivo: Crear mocks tipados para el servicio.

  1. Crea mock para el repositorio
  2. Mockea funciones externas
  3. Verifica llamadas con tipos

Código del Lab

import { TaskRepository } from "../src/repositories/taskRepository";
import { TaskService } from "../src/services/taskService";

describe("TaskService with mocks", () => {
  let mockRepository: jest.Mocked;
  let service: TaskService;
  
  beforeEach(() => {
    mockRepository = {
      findAll: jest.fn(),
      findById: jest.fn(),
      save: jest.fn(),
      delete: jest.fn()
    };
    service = new TaskService(mockRepository);
  });
  
  test("getAll calls repository", () => {
    mockRepository.findAll.mockReturnValue([]);
    const result = service.getAll();
    expect(mockRepository.findAll).toHaveBeenCalled();
    expect(result).toEqual([]);
  });
  
  test("getAll returns tasks from repository", () => {
    const mockTasks: Task[] = [
      { id: "1", title: "Task 1", status: "pending", priority: "high" }
    ];
    mockRepository.findAll.mockReturnValue(mockTasks);
    const result = service.getAll();
    expect(result).toEqual(mockTasks);
  });
});

5. SpyOn y toThrow

SpyOn

Permite espiar métodos específicos de un objeto sin reemplazar toda la implementación.

const consoleSpy = vi.spyOn(console, "log");

console.log("test");
expect(consoleSpy).toHaveBeenCalledWith("test");

consoleSpy.mockRestore();

toThrow

Verifica que una función lance un error.

function validateAge(age: number): void {
  if (age < 0) {
    throw new Error("Age cannot be negative");
  }
}

test("throws error for negative age", () => {
  expect(() => validateAge(-1)).toThrow("Age cannot be negative");
});

Lab: Tests de Error y Spies

Objetivo: Implementar tests con spies y verificación de errores.

  1. Usa vi.spyOn para espiar llamadas

Código del Lab

// Test con spyOn
test("calls logger on error", () => {
  const loggerSpy = vi.spyOn(logger, "error");
  
  service.processTask(task);
  
  expect(loggerSpy).toHaveBeenCalled();
  loggerSpy.mockRestore();
});

// Test con toThrow
test("throws when task not found", () => {
  expect(() => service.deleteTask("non-existent"))
    .toThrow("Task not found");
});

// Spy con mockImplementation
test("mocks Date.now()", () => {
  const mockDate = new Date("2024-01-01");
  vi.spyOn(global, "Date").mockImplementation(() => mockDate);
  
  const result = service.getTimestamp();
  expect(result).toBe(mockDate.getTime());
});

Tips y Gotchas

✅ Tips

  • Arrange-Act-Assert (AAA) estructura tests claros
  • Tests unitarios: prueba una cosa, un assertion
  • Mocks tipados con vi.fn() y vi.mock()

⚠️ Gotchas

  • any en mocks defeatiendo el propósito - tipa correctamente
  • Tests asíncronos necesitan async/await
  • Coverage no garantiza tests correctos, solo cantidad
