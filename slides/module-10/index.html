1. Contexto

Organización del Código

Por qué Módulos

Los módulos organizan el código en unidades reutilizables:

src/
├── types/
│   └── index.ts      # Tipos compartidos
├── services/
│   └── taskService.ts # Lógica de negocio
├── utils/
│   └── helpers.ts    # Utilidades
└── index.ts          # Punto de entrada

2. Estructura de Módulos

Módulos en TypeScript

Los módulos organizan el código en archivos independientes con export e import. 

// src/types/index.ts
export type TaskStatus = "pending" | "inProgress" | "completed";

export interface Task {
  id: string;
  title: string;
  status: TaskStatus;
}

export interface User {
  id: string;
  name: string;
}
// src/services/taskService.ts
import { Task, TaskStatus } from "../types";

export class TaskService {
  private tasks: Task[] = [];
  
  addTask(task: Task): void {
    this.tasks.push(task);
  }
  
  getTasks(): Task[] {
    return [...this.tasks];
  }
}

3. Export e Import Avanzado

export default vs export named y Re-exports

export default para una sola exportación principal; export named para múltiples. Re-exports simplifican barrel files. 

// export named: varios exports
export type TaskStatus = "pending" | "inProgress";
export interface Task { id: string; title: string; }

// export default: un solo export principal
export default class TaskService { }

// Re-export desde barrel
export { Task, TaskStatus } from "./types";

import type

import type solo importa tipos, eliminando el código en runtime. 

// Solo tipos: se elimina en compilación
import type { Task, User } from "./types";

// Import mixto (valor + tipo)
import { Task } from "./types";
import type { TaskStatus } from "./types";

3. Rutas y Paths

Path Mapping

Configura rutas limpias en tsconfig.json: 

{
  "compilerOptions": {
    "baseUrl": "./src",
    "paths": {
      "@types/*": ["types/*"],
      "@models/*": ["models/*"],
      "@services/*": ["services/*"],
      "@utils/*": ["utils/*"]
    }
  }
}

Uso:

import { Task } from "@types";
import { TaskModel } from "@models/task";
import { TaskService } from "@services/taskService";

Resumen

Puntos Clave

  • Los módulos organizan el código en unidades reutilizables
  • export hace disponible código fuera del módulo
  • import consume código de otros módulos
  • Paths permiten imports más limpios
  • baseUrl define la ruta raíz para los paths

Tips y Gotchas

✅ Tips

  • Path mapping (paths en tsconfig) evita imports relativos largos
  • Barrel files (index.ts) simplifican imports
  • Un archivo = una responsabilidad

⚠️ Gotchas

  • Path mapping solo funciona en compilación, no en runtime
  • Evita imports circulares
  • baseUrl debe apuntar al root del proyecto
