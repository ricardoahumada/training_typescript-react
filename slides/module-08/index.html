2. Clases Básicas

Clases con Tipos

Las clases combinan datos (propiedades) y comportamiento (métodos) en una sola unidad con tipos.

class Task {
  // Propiedades tipadas
  id: string;
  title: string;
  private _status: string = "pending";
  
  // Constructor
  constructor(id: string, title: string) {
    this.id = id;
    this.title = title;
  }
  
  // Getter
  get status(): string {
    return this._status;
  }
  
  // Método
  complete(): void {
    this._status = "completed";
  }
}

const task = new Task("1", "Nueva tarea");
task.complete();

Propiedades readonly y static

readonly impede modificaciones; static permite acceso sin instanciar.

class Config {
  readonly version: string = "1.0";
  static instance: Config | null = null;

  private constructor() {}

  static getInstance(): Config {
    if (!Config.instance) {
      Config.instance = new Config();
    }
    return Config.instance;
  }
}

const config = Config.getInstance();
console.log(config.version);

Parameter Properties

El constructor puede declarar propiedades directamente.

class User {
  constructor(
    public name: string,
    private id: string
  ) {}
}

Getters y Setters

Permiten acceso controlado a propiedades.

class Task {
  private _status: string = "pending";

  get status(): string {
    return this._status;
  }

  set status(value: string) {
    if (!["pending", "inProgress", "completed"].includes(value)) {
      throw new Error(`Estado inválido`);
    }
    this._status = value;
  }
}

Código del Lab

class TaskManager {
  private tasks: Task[] = [];
  
  addTask(title: string, priority: TaskPriority): Task {
    const task = createTask(title, priority);
    this.tasks.push(task);
    return task;
  }
  
  getTasks(): Task[] {
    return [...this.tasks];
  }
  
  findTaskById(id: string): Task | undefined {
    return this.tasks.find(t => t.id === id);
  }
  
  removeTask(id: string): boolean {
    const index = this.tasks.findIndex(t => t.id === id);
    if (index > -1) {
      this.tasks.splice(index, 1);
      return true;
    }
    return false;
  }
  
  getStats(): { total: number; byStatus: Record<TaskStatus, number> } {
    const byStatus: Record<TaskStatus, number> = {
      pending: 0,
      inProgress: 0,
      completed: 0
    };
    
    this.tasks.forEach(task => {
      byStatus[task.status]++;
    });
    
    return { total: this.tasks.length, byStatus };
  }
}

3. Interfaces

Definiendo Contratos

Una interface define qué propiedades y métodos debe tener una clase, sin implementación.

// Interface para repositorio
interface ITaskRepository {
  findAll(): Task[];
  findById(id: string): Task | null;
  save(task: Task): Task;
  delete(id: string): boolean;
}

// Implementación
class InMemoryTaskRepository implements ITaskRepository {
  private tasks: Task[] = [];
  
  findAll(): Task[] {
    return [...this.tasks];
  }
  
  findById(id: string): Task | null {
    return this.tasks.find(t => t.id === id) || null;
  }
  
  save(task: Task): Task {
    const existing = this.tasks.findIndex(t => t.id === task.id);
    if (existing > -1) {
      this.tasks[existing] = task;
    } else {
      this.tasks.push(task);
    }
    return task;
  }
  
  delete(id: string): boolean {
    const index = this.tasks.findIndex(t => t.id === id);
    if (index > -1) {
      this.tasks.splice(index, 1);
      return true;
    }
    return false;
  }
}

Propiedades readonly y ?

readonly marca una propiedad como inmutable; ? la hace opcional.

interface User {
  readonly id: string;
  name: string;
  email?: string;
}

// Error: no se puede asignar a propiedad readonly
// user.id = "2"; // Error!

Herencia de Interfaces (extends)

Una interface puede extender a otra, heredando sus propiedades.

interface Animal {
  name: string;
}

interface Pet extends Animal {
  ownerId: string;
}

interface Dog extends Pet {
  breed: string;
}

Index Signatures

Permite acceder a propiedades con claves dinámicas.

interface StringMap {
  [key: string]: string;
}

const strings: StringMap = {
  key1: "value1",
  key2: "value2"
};

Lab: Interfaz de Servicio

Objetivo: Crear interfaz de servicio para el sistema de tareas.

  1. Define la interfaz ITaskService
  2. Implementa la clase TaskServiceImpl
  3. Usa la interfaz en lugar de la implementación

Código del Lab

interface ITaskService {
  create(title: string, priority: TaskPriority): Task;
  getAll(): Task[];
  getById(id: string): Task | null;
  update(task: Task): Task | null;
  delete(id: string): boolean;
  filterByStatus(status: TaskStatus): Task[];
  filterByPriority(priority: TaskPriority): Task[];
}

class TaskService implements ITaskService {
  constructor(private repository: ITaskRepository) {}
  
  create(title: string, priority: TaskPriority): Task {
    const task = createTask(title, priority);
    return this.repository.save(task);
  }
  
  getAll(): Task[] {
    return this.repository.findAll();
  }
  
  getById(id: string): Task | null {
    return this.repository.findById(id);
  }
  
  update(task: Task): Task | null {
    const existing = this.repository.findById(task.id);
    if (!existing) return null;
    return this.repository.save(task);
  }
  
  delete(id: string): boolean {
    return this.repository.delete(id);
  }
  
  filterByStatus(status: TaskStatus): Task[] {
    return this.getAll().filter(t => t.status === status);
  }
  
  filterByPriority(priority: TaskPriority): Task[] {
    return this.getAll().filter(t => t.priority === priority);
  }
}

4. Herencia

Herencia de Clases

extends permite crear una clase hija que hereda propiedades y métodos de una clase padre.

// Clase base
class Person {
  constructor(public name: string) {}
  greet(): string {
    return `Hola, soy ${this.name}`;
  }
}

// Clase derivada
class Employee extends Person {
  constructor(name: string, public employeeId: string) {
    super(name);
  }
  
  greet(): string {
    return `${super.greet()} y soy empleado #${this.employeeId}`;
  }
}

const emp = new Employee("Juan", "E001");
emp.greet(); // "Hola, soy Juan y soy empleado #E001"

Method Overriding y super

super permite invocar el método de la clase padre; override reescribe el comportamiento.

class Base {
  greet(): string {
    return "Hola";
  }
}

class Derived extends Base {
  greet(): string {
    return super.greet() + ", mundo!";
  }
}

const d = new Derived();
d.greet(); // "Hola, mundo!"

Lab: Tipos Especiales de Tareas

Objetivo: Implementar herencia para tipos especializados de tareas.

  1. Crea clase base abstracta BaseTask
  2. Crea clase RegularTask que hereda de BaseTask
  3. Crea clase UrgentTask que hereda de BaseTask
  4. Crea clase RecurringTask que hereda de BaseTask

Código del Lab

abstract class BaseTask {
  constructor(
    public id: string,
    public title: string,
    public status: TaskStatus = "pending"
  ) {}
  
  abstract getPriority(): number;
  
  complete(): void {
    this.status = "completed";
  }
}

class RegularTask extends BaseTask {
  constructor(
    id: string,
    title: string,
    public priority: TaskPriority
  ) {
    super(id, title);
  }
  
  getPriority(): number {
    const levels: Record = {
      low: 1, medium: 2, high: 3
    };
    return levels[this.priority];
  }
}

class UrgentTask extends BaseTask {
  constructor(
    id: string,
    title: string,
    public reason: string
  ) {
    super(id, title);
  }
  
  getPriority(): number {
    return 4; // Siempre alta prioridad
  }
}

class RecurringTask extends BaseTask {
  constructor(
    id: string,
    title: string,
    public frequency: "daily" | "weekly" | "monthly",
    public completedCount: number = 0
  ) {
    super(id, title);
  }
  
  complete(): void {
    super.complete();
    this.completedCount++;
  }
  
  getPriority(): number {
    return this.completedCount === 0 ? 3 : 1;
  }
}

Resumen

Puntos Clave

  • public: accesible desde cualquier lugar
  • private: solo accesible dentro de la clase
  • protected: accesible desde la clase y sus subclases
  • interfaces definen contratos sin implementación
  • extends para herencia de clases
  • implements para implementar interfaces

Tips y Gotchas

✅ Tips

  • interface para contratos que pueden implementarse
  • type para uniones, interserciones y alias
  • implements es para interfaces, extends para clases

⚠️ Gotchas

  • abstract classes no se pueden instanciar directamente
  • Multiple interfaces: class A implements B, C
  • protected permite acceso en subclases, private no
