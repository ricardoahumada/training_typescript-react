1. Contexto

Funciones en TypeScript

El Problema con JavaScript

// Sin tipos, sin documentación
function processUser(data) {
  return {
    id: data.id,
    name: data.name.toUpperCase(),
    active: data.active || true
  };
}

¿Qué pasa si data.name es undefined? ¿Qué tipo retorna la función?

La Solución con TypeScript

interface UserInput {
  id: string;
  name: string;
  active?: boolean;
}

interface UserOutput {
  id: string;
  name: string;
  active: boolean;
}

function processUser(data: UserInput): UserOutput {
  return {
    id: data.id,
    name: data.name.toUpperCase(),
    active: data.active ?? true
  };
}

2. Tipos de Parámetros

Parámetros Tipados

Los parámetros pueden ser obligatorios, opcionales o tener valores por defecto.

// Parámetros con tipos explícitos
function greet(name: string): string {
  return `Hola, ${name}!`;
}

// Múltiples parámetros
function createUser(name: string, age: number, active: boolean) {
  return { name, age, active };
}

// Parámetros opcionales (con ?)
function findUser(id: string, includeProfile?: boolean) {
  // includeProfile puede ser undefined
}

// Parámetros con valores por defecto
function paginate(items: string[], page: number = 1, limit: number = 10) {
  const start = (page - 1) * limit;
  return items.slice(start, start + limit);
}

// Rest parameters
function sum(...numbers: number[]): number {
  return numbers.reduce((a, b) => a + b, 0);
}

Código del Lab

type TaskStatus = "pending" | "inProgress" | "completed";
type TaskPriority = "low" | "medium" | "high";

interface Task {
  id: string;
  title: string;
  status: TaskStatus;
  priority: TaskPriority;
  dueDate?: Date;
}

function createTask(
  title: string,
  priority: TaskPriority = "medium"
): Task {
  return {
    id: crypto.randomUUID(),
    title,
    status: "pending",
    priority
  };
}

function updateTaskStatus(
  task: Task,
  newStatus: TaskStatus
): Task {
  return {
    ...task,
    status: newStatus,
    ...(newStatus === "completed" ? { completedAt: new Date() } : {})
  };
}

function getTasksByStatus(
  tasks: Task[],
  status: TaskStatus
): Task[] {
  return tasks.filter(task => task.status === status);
}

3. Tipos de Función

Function Types

Los function types definen la firma de una función para usarla como tipo.

// Tipo de función
type MathOperation = (a: number, b: number) => number;

const add: MathOperation = (a, b) => a + b;
const multiply: MathOperation = (a, b) => a * b;

// Callbacks tipados
function processItems(
  items: T[],
  transform: (item: T) => T
): T[] {
  return items.map(transform);
}

const result = processItems([1, 2, 3], (x) => x * 2);

4. Function Overloads

Overloads

Los overloads permiten múltiples firmas para una misma función.

// Overloads
function parseDate(date: string): Date;
function parseDate(date: number): Date;
function parseDate(date: string | number): Date {
  if (typeof date === "string") {
    return new Date(date);
  }
  return new Date(date);
}

// Ejemplo con diferentes retornos
function getUser(id: string): User;
function getUser(email: string): User | null;

Lab: API de Tareas con Overloads

Objetivo: Implementar función de búsqueda con múltiples firmas.

  1. Implementa overloads para findTask

Código del Lab

// Sobrecargas
function findTask(id: string): Task | undefined;
function findTask(title: string, exactMatch: boolean): Task[];
function findTask(
  idOrTitle: string,
  exactMatch: boolean = true
): Task | Task[] | undefined {
  if (exactMatch) {
    return tasks.find(t => t.id === idOrTitle);
  } else {
    return tasks.filter(t => 
      t.title.toLowerCase().includes(idOrTitle.toLowerCase())
    );
  }
}

Resumen

Puntos Clave

  • Los parámetros pueden ser obligatorios, opcionales o tener valores por defecto
  • Los rest parameters usan el operador ...

  • Overloads permiten múltiples firmas

Tips y Gotchas

✅ Tips

  • Declara tipos explícitos para parámetros y retornos siempre
  • ? para opcionales, = para valores por defecto
  • Function overloads: firma más específica primero

⚠️ Gotchas

  • Arrow functions capturan this
  • Parámetros por defecto pueden ser undefined
  • never retornan implícitamente si lanzas errores
